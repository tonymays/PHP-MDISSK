Audit Log

The Audit Log report allows a user to review audited changes that have occurred within the system as its usage has evolved. The Audit Log report is a multiple instance window and can be loaded as many times as needed.

Do the following to search for audited changes:

1 - Select the user from the Users drop down; or, select All for all users
2 - Select the module to examine from the Modules drop down; or, select All for all modules
3 - Select a query start date by clicking on the calendar next to the Start Date input box
4 - Select a query end date by clicking on the calendar next to the End Date input box
5 - And finally, click the Search Button

Do the following to reset the filters for a new search:

1 - Click the Reset button
2 - Then follow the steps outlined above for searching for audited changes

Please note:

1 - The Start Date is non editable unless you want to clear a date
1 - The End Date is non editable unless you want to clear a date

Report notes: Once a report has been ran with output, you can do the following:

1 - Click on the user link under User Name to view the user's profile that made the change
2 - Click on the module line under Module to view what was changed by the submitting transaction
