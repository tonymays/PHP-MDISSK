Permissions

The Permission page allows an administrator to establish specific rights, or permissions, within the system for a specified role.

Do the following to set or unset a permission:

1 - Select the targeted role from the Roles drop down
2 - Check or un-check the Permit checkbox for the specified permission sought
3 - Repeat step 2 until all permissions required have been made
4 - And finally, click the Save button

Column Descriptions:

The Permission page is group by module for the specified role. There are 4 columns of information: Submodule, Description, Permission Type and Permit. The Submodule and Description columns give a brief description of an individual permission set. There are two types of Permissions: Feature and Page Permission. A Feature Permission determines what will be displayed in the Site Menu and represents a feature of the site. A Page Permission determines what rights a user has on a specific page once loaded.

Page permissions that are set to on while there corresponding Feature permissions are set to off will have no affect on the system.

Please note:

Permission sets that are changed do not come into affect until the targeted user(s) logs back into the system. For permission changes to take affect immediately, ensure that the targeted user logs out and back in after a permission set change.
